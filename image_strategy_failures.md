# 이미지 처리 전략 실패 기록 및 분석 리포트

본 문서는 카페24 호스팅 용량 문제를 해결하기 위해 시도했던 외부 이미지 연동 전략들의 실패 과정과 기술적 원인을 기록합니다.

## 1. 배경
- **문제:** 카페24 웹호스팅의 적은 용량으로 인해 이미지 업로드 시 금방 한계에 도달함.
- **목표:** 이미지를 서버에 저장하지 않고 외부 URL(원본 기사 또는 이미지 호스팅)을 사용하여 썸네일(특성 이미지)을 구현함.

## 2. 시도했던 전략 및 실패 원인

### 전략 1: Cloudinary 활용
- **방법:** 이미지를 Cloudinary 서버에 올리고 그 URL을 사용.
- **결과:** 설정이 복잡하고 API 연동 단계에서 효율성이 떨어진다고 판단하여 중단.

### 전략 2: 'External URL Featured Image' 플러그인 + REST API
- **방법:** REST API의 `meta` 필드에 `_harikrutfiwu_url` 등을 넣어 전송.
- **실패 원인:** 
    - 워드프레스 보안상 `_`로 시작하는 비공개 메타 키는 REST API를 통한 수정이 기본적으로 차단됨.
    - 'WP REST API Controller'로 권한을 부여했음에도 불구하고 필드가 정상적으로 업데이트되지 않음.

### 전략 3: 'FIFU' 플러그인 + SSH/WP-CLI (`wp eval`)
- **방법:** 파이썬에서 SSH로 서버에 접속하여 PHP 함수 `fifu_dev_set_image()`를 직접 호출.
- **실패 원인:**
    - **따옴표 충돌:** 윈도우 파워쉘에서 명령어를 보낼 때 이미지 URL의 특수문자(`&`, `?`, `=`)가 깨지거나 PHP 문법 에러를 유발함.
    - **환경 미비:** 카페24 기본 환경에서 `php`, `wp` 명령어를 찾지 못해 절대 경로 지정이 필요했음.

### 전략 4: 'FIFU' 플러그인 + SSH/WP-CLI (`wp post meta update`)
- **방법:** 워드프레스 표준 메타 업데이트 명령어를 사용하여 `fifu_image_url`과 `_fifu_image_url`을 강제 주입.
- **결과:** 데이터베이스 업데이트에는 성공(`SUCCESS` 출력)했으나, 실제 웹사이트 상에서 이미지가 렌더링되지 않는 현상 발생. (플러그인 내부의 추가적인 프로세싱이 누락된 것으로 추정)

## 3. 핵심 기술적 장애물 (Lessons Learned)

1. **윈도우 쉘 이스케이프 문제:** 
   Windows 환경에서 `subprocess`를 통해 SSH 명령을 날릴 때, 이중/삼중 따옴표 처리가 리눅스 서버와 달라 URL이 변조됨. 이를 해결하기 위해 Base64 인코딩을 시도했으나 PHP 레이어에서 다시 해석 에러 발생.

2. **인코딩 이슈:**
   윈도우의 `cp949` 인코딩과 서버의 `utf-8` 응답이 충돌하여 파이썬 프로세스가 결과를 읽어오는 도중 `UnicodeDecodeError` 발생.

3. **워드프레스 내부 로직:**
   단순히 DB의 메타 값만 바꾼다고 해서 썸네일이 즉시 나타나지 않음. 워드프레스 엔진이 해당 이미지를 '특성 이미지'로 정식 등록하는 별도의 Hook 과정이 필요함.

## 4. 현재 상태
- **결정:** 모든 외부 이미지 연동 시도를 중단하고, 가장 안정적인 **'직접 업로드(A1 버전)'** 방식으로 롤백함.
- **보완:** 카테고리를 **ID 21(News)**로 고정하여 분류 정확도를 높임.

## 5. 추후 권장 접근법
- `register_post_meta`를 `functions.php`에 등록하여 표준 REST API 가이드를 따르는 방식.
- 이미지를 업로드하기 전 파이썬 `Pillow` 라이브러리로 용량을 90% 이상 압축(WebP 변환)하여 업로드하는 방식.
